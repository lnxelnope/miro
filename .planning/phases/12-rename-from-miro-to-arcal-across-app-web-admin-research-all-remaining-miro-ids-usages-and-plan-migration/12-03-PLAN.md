## Migration Strategy

สำหรับ IDs/slug ที่เลือก action = rename-now หรือ create-alias:
- ระบุลำดับขั้นตอน migration ทีละข้อ:
  1. เพิ่ม field/column ใหม่หรือ topic ใหม่ (arcal_*)
  2. ทำ dual-write ชั่วคราว (เขียนข้อมูลลงทั้ง miro_* และ arcal_*)
  3. migrate data เก่า (backfill)
  4. เปลี่ยน consumer ให้ไปอ่านจาก arcal_* แทน
  5. เก็บ metrics/alert ดู regression
  6. ปิดการใช้งาน miro_* เมื่อผ่านระยะเวลาปลอดภัย

หากใน v2.0 ยังไม่ทำ migration จริง ให้บันทึกว่า "Out of scope for v2.0 — keep legacy IDs" และใส่ link ไป future phase/milestone ที่จะตามต่อ
  </action>
  <verify>มีแผน migration แบบ step-by-step สำหรับอย่างน้อย IDs ที่ critical/high-risk</verify>
