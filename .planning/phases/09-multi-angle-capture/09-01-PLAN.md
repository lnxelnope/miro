```dart
enum AngleZone {
  top,      // 0° ±15° (มองลงตรง ๆ)
  diagonal, // 45° ±20°
  side,     // 70° ±20°
  outOfRange, // มุมไม่ตรงกับ zone ใด
}

class AngleCaptureResult {
  final AngleZone zone;
  final String imagePath;       // path ที่เก็บรูปบนดิสก์
  final DateTime capturedAt;
  final double actualAngle;     // มุมจริงที่วัดได้ตอน capture
}
```

เพิ่ม helper method `AngleZone.fromDegrees(double degrees)` ที่:
- 0-15° → top
- 25-65° → diagonal  
- 50-90° → side
- อื่น ๆ → outOfRange

(ranges ซ้อนกันเล็กน้อยได้ เน้นให้ผู้ใช้ไม่ต้องจัดมุมเป๊ะ)

สร้าง class ที่:
1. Subscribe `accelerometerEventStream()` จาก `sensors_plus` package
2. คำนวณ pitch angle (มุมเอียงกล้อง) จาก accelerometer values:
   ```dart
   // pitch = มุมระหว่างแนวตั้งกับกล้อง (0° = มองลงตรง, 90° = มองตรงหน้า)
   double pitch = atan2(y, sqrt(x*x + z*z)) * (180 / pi);
   double cameraAngle = (90 - pitch.abs()).clamp(0, 90);
   ```
3. Apply low-pass filter (smoothing factor ~0.2) เพื่อลด noise:
   ```dart
   _smoothedAngle = _smoothedAngle * 0.8 + rawAngle * 0.2;
   ```
4. Expose `ValueNotifier<double> currentAngle` (0-90°) และ `ValueNotifier<AngleZone> currentZone`
5. มี method `bool isStableInZone(AngleZone zone, Duration requiredDuration)` — ตรวจว่ามุมอยู่ใน zone เดิมนานพอหรือยัง (ใช้ timestamp เปรียบเทียบ)
6. มี `dispose()` สำหรับ cancel subscription

  <action>
สร้าง controller ที่:

**State ที่ expose (ใช้ ValueNotifier ทั้งหมด):**
```dart
ValueNotifier<int> capturedCount = ValueNotifier(0);  // 0-3
ValueNotifier<AngleZone?> currentTargetZone;           // มุมที่ต้องถ่ายต่อไป
ValueNotifier<AngleZone?> currentDeviceZone;            // มุมปัจจุบัน
ValueNotifier<double> currentAngle;                     // มุมองศาปัจจุบัน
ValueNotifier<AngleCaptureResult?> lastCaptureResult;   // ผล capture ล่าสุด
ValueNotifier<bool> isCapturing;                        // กำลัง capture อยู่
List<AngleCaptureResult> capturedImages = [];           // ผลลัพธ์ทั้ง 3 มุม
```

**Flow logic:**
1. เริ่มที่ targetZone = top (0°) → diagonal (45°) → side (70°) ตามลำดับ
2. เมื่อ sensor อยู่ใน target zone + stable ≥ 0.7 วิ + detection state == foodFound/readyForCapture → trigger auto-capture
3. Auto-capture process (สำคัญมาก):
   ```
   ⚠️ CRITICAL CONSTRAINT: Flutter camera plugin ไม่สามารถ startImageStream + takePicture พร้อมกันได้
   ต้อง: stopImageStream() → takePicture() → startImageStream()
   ```
   a. Set isCapturing = true
   b. เรียก `cameraController.stopImageStream()`
   c. เรียก `cameraController.takePicture()` → ได้ XFile
   d. Save รูปไปที่ app temp dir: `getTemporaryDirectory()/arscan_{timestamp}_{zone.name}.jpg`
   e. เรียก `cameraController.startImageStream(onFrame)` กลับมา
   f. สร้าง AngleCaptureResult, เพิ่มเข้า capturedImages, อัปเดต capturedCount
   g. Set isCapturing = false, ย้าย targetZone ไป zone ถัดไป
4. ถ้า capturedCount == 3 → set state เป็น complete (Phase 10 จะ listen)

**Manual shutter:**
- เมื่อผู้ใช้กด manual → ถ่ายทันทีไม่ว่า sensor จะอยู่ zone ไหน
- ถ้ามุมปัจจุบันตรงกับ zone ที่ถ่ายไปแล้ว → ทับรูปเดิม (replace ใน capturedImages)
- ถ้า auto กำลังจะ trigger อยู่ → cancel auto, ใช้ manual แทน

**Reset เมื่อกล้องหนีจากอาหาร (ARSCAN-04):**
- ถ้า detection primary box หายไป (ArScanState == searching) นานเกิน 1 วินาที:
  - reset capturedCount = 0, clear capturedImages, ลบไฟล์รูปเก่า
  - กลับไปเริ่ม targetZone = top
  - ไม่ต้องถาม/ขึ้น dialog
  </action>
