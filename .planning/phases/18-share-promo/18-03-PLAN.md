# Plan 18-03: Settings — redeem promo + client integration

## Objective

ส่งมอบ **ARC2-PROMO-01** ตาม `18-CONTEXT` **D-06–D-08**: ช่องกรอกโค้ดใน **Settings** (`ProfileScreen`), เรียก endpoint จากแผน **18-02**, แจ้งผล **SnackBar สั้นๆ** เมื่อสำเร็จ และข้อความ L10n เมื่อล้มเหลว

## Context

- **REQ:** ARC2-PROMO-01
- **Depends:** แผน 18-02 deploy / พร้อมทดสอบ

## Requirements

1. **UI:** การ์ดหรือส่วนย่อยใน `ProfileScreen` — ช่อง `TextField` โค้ด, ปุ่ม Redeem, รองรับ keyboard done
2. **Service:** เช่น `PromoCodeService` หรือ method ใน `EnergyService` — `POST` ไป `.../redeemPromoCode` พร้อม `deviceId` (และฟิลด์อื่นที่ backend กำหนด)
3. **หลังสำเร็จ:** **SnackBar สั้น** (L10n) — **D-07**; เรียก `syncBalanceWithServer` / `registerOrSync` หรือ invalidate providers ที่อัปเดต energy & freepass ให้ UI สะท้อนทันที
4. **หลังล้มเหลว:** แปลง body/error จาก API เป็น key ข้อความ (ไม่แสดง raw stack ให้ผู้ใช้)
5. **L10n:** ทุกข้อความผู้ใช้จาก ARB
6. **Analytics (optional):** event redeem success/fail โดยไม่ส่งโค้ดเต็ม

## Tasks

### Task 1: Service layer

- [ ] Implement HTTP client สอดคล้อง `EnergyService` (URL base, JSON body, timeout)
- [ ] Parse response: balance / freepass / message

### Task 3: State / loading

- [ ] ปุ่ม loading ขณะ redeem; กัน double-tap

### Task 4: Verify
