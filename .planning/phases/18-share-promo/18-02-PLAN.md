# Plan 18-02: Promo backend + admin-panel CRUD

## Objective

ส่งมอบส่วน **ARC2-PROMO-02** ฝั่งเซิร์ฟเวอร์และแอดมิน: เก็บโค้ด promo, จำกัด **ครั้งรวม** / **ต่อผู้ใช้** / **วันหมดอายุ**, และ endpoint **redeem** ที่ idempotent พอใช้งานจริง — ตาม `18-CONTEXT` **D-09, D-10, D-08** (ข้อความ error ฝั่ง API)

## Context

- **REQ:** ARC2-PROMO-02 (และ foundation สำหรับ ARC2-PROMO-01)
- **ต่อยอด:** `functions/` + `admin-panel/` ใน repo เดียวกัน — **ไม่** สร้าง stack ใหม่

## Requirements

1. **Firestore (หรือ store ที่โปรเจกต์ใช้):** collection / โครงเอกสาร promo code — ฟิลด์ขั้นต่ำ: `code` (normalized uppercase), `active`, `rewardType`, `rewardPayload` (เช่น energy amount, freepass days), `maxRedemptions`, `perUserLimit`, `expiryAt`, `redemptionCount`, `createdAt`, `updatedAt`
2. **Cloud Function (HTTPS onRequest หรือแพทเทิร์นเดียวกับ endpoint อื่นใน repo):** `redeemPromoCode` — รับตัวระบุผู้ใช้แบบเดียวกับ `registerUser` / `syncBalance` (`deviceId` + โครงสร้างที่มีอยู่) + สตริงโค้ด — ตรวจโค้ด, expiry, limits, แล้วอัปเดตยอด energy หรือ freepass ตาม `rewardType` — **กันยิงซ้ำ** (เช่น transaction / idempotency key หรือบันทึก redemption ต่อ user+code)
3. **ความปลอดภัย:** ไม่ log โค้ดเต็มใน production; rate limit พื้นฐานถ้าทำได้ต่ำต้นทุน
4. **admin-panel:** หน้า list + create + edit promo codes (คัดลอกแพทเทิร์นจาก `offers` — `src/app/(dashboard)/offers/`, `OfferForm.tsx`) — เรียก API ภายใน admin ที่อ่าน/เขียน Firestore หรือเรียก Functions admin-only ตามที่โปรเจกต์ทำกับ offers
5. **Export:** ลงทะเบียน function ใน `functions/src/index.ts`
