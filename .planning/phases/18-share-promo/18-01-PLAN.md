# Plan 18-01: Share card presets + food-only serving on image

## Objective

ส่งมอบ **ARC2-SHARE-01** ตาม `18-CONTEXT` **D-01–D-05**: ผู้ใช้เลือก preset **16:9** หรือ **1:1** ได้สำหรับ **ทุก** `ShareCardType`; toggle **แสดงปริมาณบนภาพ** มีเฉพาะ **food item** card

## Context

- **REQ:** ARC2-SHARE-01
- **ล็อกจาก discuss:** toggle ปริมาณไม่ใช้กับ daily / nutrition

## Requirements

1. เพิ่ม enum หรือค่าที่เทียบเท่า **`ShareCardAspect`** (เช่น `ratio16x9`, `ratio1x1`) ใน `ShareCardConfig` + `copyWith`
2. **ทั้งสาม** widget การ์ด (`ShareCardFoodItem`, `ShareCardDailySummary`, `ShareCardNutritionSummary`) วาดภายในกรอบที่สอดคล้อง aspect ที่เลือก (อัตราส่วนคงที่ก่อน capture)
3. **`ShareCardCreatorScreen`:** UI เลือก aspect (segmented / chips — discretion) สำหรับทุกประเภทการ์ด
4. **`showServingOnImage`:** แสดงใน UI เฉพาะเมื่อ `_config.type == ShareCardType.foodItem`; เมื่อเปิด ให้แสดงข้อความปริมาณระดับ entry บนการ์ด (L10n + ตัวเลขจาก `FoodEntry` + หน่วยผ่าน `UnitConverter` / แสดงผลที่สอดคล้อง `.cursorrules`)
5. ถ้าไม่มี serving ที่ใช้แสดงได้ — ซ่อนหรือ disable toggle พร้อมข้อความสั้น (L10n) ตาม CONTEXT discretion
6. **`CardCaptureService`:** capture จาก key เดิม โดยผลลัพธ์สะท้อน pixel ratio / ขนาดที่เหมาะกับโซเชียล (discretion — เช่น 1920×1080 vs 1080×1080 logical mapping)
7. Analytics: ส่งพารามิเตอร์ `aspect` (และ `showServing` สำหรับ food) ใน event ที่มีอยู่ถ้าทำได้ง่าย

## Tasks

### Task 1: Model + L10n

- [ ] ขยาย `ShareCardConfig` — aspect + `showServingOnImage` (default false)
- [ ] เพิ่ม keys ARB สำหรับป้าย aspect, ปริมาณบนการ์ด, กรณีไม่มี serving

### Task 2: Layout การ์ดสามแบบ

- [ ] ห่อเนื้อหาด้วย `AspectRatio` หรือ `SizedBox` คงที่ตาม preset — ปรับ gradient/รูป/ข้อความไม่ให้ถูก crop ผิด
- [ ] `ShareCardFoodItem`: เมื่อ `showServingOnImage` แสดงบรรทัดปริมาณตาม D-02

### Task 3: Creator screen + capture

- [ ] ผูก state `_config` กับตัวเลือก aspect + toggle (food เท่านั้น)
- [ ] อัปเดต `CardCaptureService` ถ้าต้องส่ง `pixelRatio` / ขนาดจากภายนอก

### Task 4: Verify

- [ ] `flutter analyze` บนไฟล์ที่แตะ
