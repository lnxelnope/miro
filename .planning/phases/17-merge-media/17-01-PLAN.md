# Plan 17-01: Group merge — selection bar + merged FoodEntry

## Objective

เพิ่มปุ่ม **Group** บน selection bar ของ `FoodSandbox` (หลัง Phase 15) และเชื่อม `BasicModeTab` ให้รวมหลาย `FoodEntry` เป็น **หนึ่งรายการ** ชื่อ `group_<timestamp>` พร้อม **`ingredientsJson` v2** ที่รวมโภชนาการและโครงสร้างตาม `17-CONTEXT` D-04–D-05, D-06, D-08

## Context

- **REQ:** ARC2-GR-01
- **Gate:** Phase 15 ต้องมี selection bar แล้ว; Phase 13+16 ต้องมี codec v2 ใช้งานได้
- **Pattern ลบหลายรายการ:** `basic_mode_tab.dart` → `_deleteSelectedEntries` (ยืนยัน → loop delete → `refreshFoodProviders`)

## Requirements

1. ปุ่ม Group บน `FoodSandbox` เมื่อ `selectedCount >= 2` (และไม่ disable ตามข้อ 2)
2. ถ้า **อย่างน้อยหนึ่ง** entry ที่เลือกมี `isGroupOriginal == true` (หรือตาม heuristic ที่ CONTEXT อนุญาต) → **disable** Group + tooltip/snackbar อธิบาย (L10n)
3. กด Group → ยืนยัน (reuse pattern `AlertDialog` แบบ delete) → สร้าง `FoodEntry` ใหม่:
   - `foodName`: `group_<timestamp>` (หรือรูปแบบเดียวกับที่ CONTEXT กำหนด)
   - แคล/โปรตีน/คาร์บ/ไขมัน = **ผลรวม** ของทุก entry ที่เลือก
   - `ingredientsJson`: **v2** — แต่ละ entry ต้นทาง = **อย่างน้อยหนึ่ง main** ใน `mainIngredients` (รายละเอียด flatten/ยุบ sub ตาม D-05)
   - `isGroupOriginal: true` บน entry ใหม่
4. หลังบันทึกสำเร็จ → **ลบ** entry เดิมที่ถูกรวม → **`refreshFoodProviders` / `invalidate` ชุดเดียวกับลบหลายรายการ** (`17-CONTEXT` D-09) + feedback (SnackBar L10n)
5. **ห้าม** nested group: ไม่ให้ merge ที่มี `isGroupOriginal` ใน selection

## Tasks

### Task 1: L10n

- [ ] เพิ่ม keys ใน `app_en.arb` / `app_th.arb`: เช่น `mergeGroup`, `confirmMergeGroup`, `groupMergeDisabledNested`, `mergedGroupSuccess` — รัน `flutter gen-l10n`

### Task 2: Builder v2 จากหลาย FoodEntry

- [ ] สร้างฟังก์ชัน/service (เช่น `lib/features/health/services/merge_food_entries.dart` หรือใน layer ที่สอดคล้องโปรเจกต์) ที่รับ `List<FoodEntry>` และคืน `(mergedTotals, ingredientsJsonString v2)` โดยใช้ `ingredients_codec` / โมเดล v2 ที่มีจาก Phase 13–16
- [ ] ครอบคลุม: entry ไม่มี ingredientsJson → ถือเป็น main เดียวต่อ entry; entry มี v1/v2 → parse แล้ว map เป็นโครง v2 ตาม CONTEXT

### Task 3: FoodSandbox — ปุ่ม Group

- [ ] เพิ่มพารามิเตอร์ callback เช่น `onMergeSelected` / `canMergeSelected` (หรือรวมใน callback เดียวกับ delete/analyze ตาม pattern ที่มี)
- [ ] แสดงปุ่ม Group เมื่อ `selectedCount >= 2` และ `canMerge == true`

### Task 4: BasicModeTab — handler

- [ ] `_mergeSelectedEntries(List<FoodEntry> entries)` — validate count ≥ 2, ไม่มี `isGroupOriginal`, เรียก builder → `addFoodEntry` → loop `deleteFoodEntry` → `refreshFoodProviders` + SnackBar
- [ ] ส่ง callback ลง `FoodSandbox`

### Task 5: Verify

- [ ] `flutter analyze` บนไฟล์ที่แตะ
- [ ] Manual: เลือก 2+ รายการ → Group → ตรวจชื่อ `group_*`, ผลรวม macro, JSON v2 ใน DB

## Success Criteria

- [ ] Group ใช้ได้เมื่อเลือก ≥2 และไม่มี nested group
- [ ] Entry ใหม่มี `isGroupOriginal: true` และ `ingredientsJson` v2 ถูกต้อง
- [ ] รายการเดิมถูกลบหลัง merge สำเร็จ
- [ ] L10n ครบ ไม่มี hardcode ข้อความผู้ใช้
