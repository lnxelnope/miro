สร้าง method ใน `_CreateMealSheetState`:

```dart
Future<void> _analyzeSubFromPhoto(_IngredientRow parentRow, int subIdx) async
```

Flow:
1. แสดง bottom sheet เลือก gallery/camera (L10n):
   - Gallery → `ImagePickerService.pickFromGallery()`
   - Camera → `ImagePickerService.pickFromCamera()`
2. ถ้าได้ `File` → `Navigator.push` ไป `ImageAnalysisPreviewScreen(imageFile: file, initialFoodName: sub.nameController.text)`
3. รอผลกลับ (ถ้า `ImageAnalysisPreviewScreen` pop กับ result — ตรวจว่า screen return อะไร; ถ้า return `FoodEntry` ให้ map; ถ้า void ต้องอ่าน pattern จากที่อื่นในแอปที่ใช้ screen นี้)
4. Map ผลกลับเข้า sub (**ตาม pattern เดียวกับ `_lookupSubIngredient` บรรทัด ~1780–1820**):
   - ตั้ง name, amount, unit, calories, protein, carbs, fat จาก result
   - `saveBaseValues()` บน sub
   - recalculate parent row
5. ตั้ง `imagePath` บน sub node (ถ้าโมเดลรองรับจากแผน 17-02)
6. `setState` เพื่ออัปเดต UI

**หมายเหตุ:** ถ้า `ImageAnalysisPreviewScreen` ไม่ return result (แค่บันทึก entry ตรง) — ต้อง **ปรับ** ให้ screen สามารถ return result กลับได้โดยไม่บันทึก (เช่น เพิ่ม mode `returnResultOnly: true`) หรือใช้ `GeminiService.analyzeFoodByImage(file)` ตรงๆ — เลือกวิธีที่ไม่ทำลาย flow อื่นที่ใช้ screen เดิม
  </action>
