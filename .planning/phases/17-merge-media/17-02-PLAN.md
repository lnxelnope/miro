# Plan 17-02: Ingredient micro thumbnail + optional image on node

## Objective

แสดง **รูปย่อขนาดเล็ก** ข้างชื่อวัตถุดิบในแถวที่เกี่ยวข้อง และรองรับ **optional `imagePath` (และ bbox ถ้าจำเป็น)** บนโหนดใน `ingredientsJson` v2 ตาม `17-CONTEXT` D-09–D-12

## Context

- **REQ:** ARC2-THUMB-01
- **Depends:** แผน 01 (โครง group + v2 คงที่); Phase 13 schema/codec

## Requirements

1. Widget **`IngredientThumb`** (หรือชื่อเดียวกันในโค้ด): กรอบสี่เหลี่ยมจัตุรัส ~24–36 logical px, `BoxFit.cover`, `ClipRRect` — **แสดงเฉพาะเมื่อมี path รูป** (`17-CONTEXT` D-10: **ไม่มี** placeholder สีเทาเมื่อไม่มีรูป)
2. ขยาย **โมเดล v2 / codec** ให้โหนด ingredient รองรับ optional `imagePath` (+ optional bbox ตามที่ CONTEXT ระบุ) — backward compatible กับข้อมูลเก่า
3. แตะ thumb → เปิดดูรูปเต็มพร้อม **overlay bbox** ถ้ามีข้อมูล (`17-CONTEXT` D-11, D-12)
4. ผูก thumb ในแถวที่แสดงรายการวัตถุดิบ เช่น:
   - Add / Edit food bottom sheet (รายการ sub/main ที่มี path)
   - Gemini analysis sheet (ถ้ามีแถว ingredient)
   - Create meal sheet — แถว sub ingredient (ร่วมกับแผน 03 ถ้าแยก commit; อย่าทำลาย `_lookupSubIngredient` — D-17)

## Tasks

### Task 1: Schema + codec

- [ ] อัปเดตโมเดล Dart v2 + parse/serialize ใน `ingredients_codec` (หรือไฟล์ที่ Phase 13 กำหนด) สำหรับ optional `imagePath` / bbox
- [ ] Migration หรือ default null — ไม่ทำลาย JSON เก่า

### Task 2: IngredientThumb widget

- [ ] สร้าง widget รับ `String? imagePath`, `double? size` — ถ้า path ว่าง return `SizedBox.shrink()` (หรือไม่ครอบพื้นที่); ถ้ามี path ใช้ `Image.file` + `errorBuilder` แบบไม่ทิ้งกล่องเทาโผล่
- [ ] ไม่ hardcode สีข้อความ — ใช้ theme / `AppColors` ที่มี

### Task 3: Full-screen preview + bbox

- [ ] เมื่อแตะ thumb → เปิด dialog / route เต็มจอ แสดงรูป; ถ้ามี bbox + ขนาดภาพ → วาด overlay (`17-CONTEXT` D-12)

### Task 4: ผูก UI

- [ ] แถว ingredient ใน `add_food_bottom_sheet`, `edit_food_bottom_sheet`, `gemini_analysis_sheet` (และ `create_meal_sheet` ถ้า scope แผนนี้รวมแถว sub) — แสดง `IngredientThumb` เมื่อมี path
- [ ] เมื่อผู้ใช้เลือกรูปสำหรับ ingredient (flow ที่มีหรือเพิ่มในแผน 03) → บันทึก path ลงโหนดก่อน serialize

### Task 5: Verify

- [ ] `flutter analyze`
- [ ] Manual: entry ที่มี path บนโหนดแสดง thumb; entry เก่าไม่ crash

## Success Criteria

- [ ] JSON v2 รองรับ optional image บนโหนด และ backward compatible
- [ ] Thumb แสดงในจุดที่กำหนด ไม่ทำให้ layout แตกบนจอเล็ก
- [ ] แตะ thumb → เต็มจอ + overlay bbox เมื่อมีข้อมูล

## Scope Boundaries

- **In scope:** widget thumb, schema field, แถวที่แสดงรายการวัตถุดิบที่มีอยู่
- **Out of scope:** flow เลือกรูปหลักของ Create Meal gallery (แผน 03) — ยกเว้นการบันทึก path ลงโหนดถ้าทำในแผน 03
